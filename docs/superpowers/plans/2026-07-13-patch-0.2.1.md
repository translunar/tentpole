### Task 1: Retry-After HTTP-date tolerance

RFC 9110 allows `Retry-After` in two forms: delta-seconds (`"7"`) and HTTP-date (`"Wed, 21 Oct 2026 07:28:00 GMT"`). `request()` currently does `float(retry_after)`, so an HTTP-date crashes the whole request with a `ValueError` mid-retry. Computing the date's delta would require a clock read, which this module deliberately avoids — so the HTTP-date form falls back to the exponential delay, which never under-waits relative to the exponential schedule.

**Files:**
- Modify: `src/tentpole/adapters/http.py:55-59`
- Test: `tests/test_adapter_http.py` (append)

**Interfaces:**
- Consumes/Produces: `request()` signature unchanged. Only the sleep-duration computation changes.

### Task 2: Token leak-proofing (`Secret` wrapper)

`repr=False` on the token fields protects `repr()`, but `dataclasses.asdict()` still returns the raw token, so any future `json.dumps(asdict(cfg))` (debug dump, log line, report) would leak it. Wrap tokens in a `Secret` that masks `repr()`/`str()`/f-strings, makes `json.dumps` raise `TypeError` (fail closed), and requires an explicit `.reveal()` at the two places that build Authorization headers. `Secret.__eq__` compares against plain strings so existing assertions (`cfg.jira.token == "jt"`) keep passing, and `__post_init__` wraps plain-string tokens so existing direct constructions (`JiraConfig(..., token="t")`) keep working.

- Modify: `src/tentpole/adapters/config.py`
- Modify: `src/tentpole/adapters/jira_extract.py` (`_headers`)
- Modify: `src/tentpole/adapters/smartsheet_load.py` (`_headers`)

**Interfaces:**
- Produces: `config.Secret` with `.reveal() -> str`, masked `__repr__`/`__str__`, `__eq__` against `str` and `Secret`, `__hash__`. `JiraConfig.token` and `SmartsheetConfig.token` are `Secret` after construction (plain `str` accepted and auto-wrapped).
- Consumers changed: `jira_extract._headers` and `smartsheet_load._headers` call `cfg.token.reveal()`. (`jira_write` reuses `jira_extract._headers` — no change there.)

```python
class Secret:
    """A credential that cannot leak through repr(), str(), f-strings,
    dataclasses.asdict(), or json.dumps() (dumps raises TypeError --
    fail closed). Call .reveal() at the moment of use, i.e. when
    building an Authorization header."""

    __slots__ = ("_value",)

    def __init__(self, value: str):
        self._value = value

    def reveal(self) -> str:
        return self._value

    def __repr__(self) -> str:
        return "Secret('***')"

    __str__ = __repr__

    def __eq__(self, other: object) -> bool:
        if isinstance(other, Secret):
            return self._value == other._value
        return self._value == other

    def __hash__(self) -> int:
        return hash(self._value)
```

### Task 4: Push pre-flights every sheet's columns before the first write

`push_plan`'s column validation currently fires per sheet inside the loop, so a mismatch on sheet 4 aborts push after sheets 1–3 were already written — the partial report is discarded and a naive re-run against a stale state dir would double-apply the earlier adds. Validate ALL target sheets' columns up front, aggregate every problem into one `ValueError`, and only then start writing. Standalone `push_plan` keeps its own validation (unchanged behavior) via an optional `col_ids` parameter that `push_plans` supplies pre-validated.

```python
def _validate_columns(schema: SheetSchema, sheet_id: int,
                      col_ids: dict) -> str | None:
    # Fail loudly but actionably (same posture as humansheets._number):
    # a column the schema expects is absent from the live sheet, so
    # every cell for it would otherwise be silently dropped from every
    # add/update, the row still tallied as a success, and the sync
    # would never converge (pull can't read back what push never
    # wrote). Refuse rather than degrade quietly.
    missing = sorted(set(schema.synced_names()) - set(col_ids))
    if not missing:
        return None
    return (f"sheet {schema.name!r} (id {sheet_id}) is missing column(s) "
            f"{missing} required by the {schema.name!r} schema; the live "
            f"sheet only has {sorted(col_ids)}. Recreate the missing "
            f"column(s) exactly as named by `tentpole schema show` before "
            f"pushing.")
```

```python
def push_plans(cfg, plans_dir: Path, state_dir: Path,
               http=request) -> dict[str, dict]:
    plans_dir, state_dir = Path(plans_dir), Path(state_dir)
    report = {}
    targets = []
    # Iterate the plans `sync` actually produced (one per machine sheet),
    # not cfg.sheets: a machine sheet missing from tentpole.yaml must be
    # reported as skipped, not silently dropped (spec: a silently failing
    # sync must be impossible).
    for name, schema in SCHEMAS.items():
        if schema.owned != "machine":
            continue
        plan_path = plans_dir / f"{name}.json"
        if not plan_path.exists():
            continue
        if name not in cfg.sheets:
            report[name] = {
                "added": 0, "updated": 0, "removed": 0,
                "failed": [{
                    "op": "push", "key": name,
                    "error": f"SKIPPED (no sheet id configured for "
                             f"{name!r} in tentpole.yaml)"}]}
            continue
        targets.append((name, schema, plan_path))

    # Pre-flight EVERY target sheet's columns before the first write: a
    # schema mismatch discovered mid-loop would abort push after earlier
    # sheets were already written, discarding the partial report and
    # leaving a re-run against a stale state dir to double-apply adds.
    col_ids_by_name = {}
    problems = []
    for name, schema, _ in targets:
        col_ids = _column_ids(cfg, cfg.sheets[name], http=http)
        problem = _validate_columns(schema, cfg.sheets[name], col_ids)
        if problem:
            problems.append(problem)
        col_ids_by_name[name] = col_ids
    if problems:
        raise ValueError("\n".join(problems))

    for name, schema, plan_path in targets:
        changes = json.loads(plan_path.read_text())
        state_path = state_dir / f"{name}.json"
        state = (json.loads(state_path.read_text())
                 if state_path.exists() else {})
        report[name] = push_plan(cfg, cfg.sheets[name], changes, state,
                                 schema, http=http,
                                 col_ids=col_ids_by_name[name])
    return report
```
